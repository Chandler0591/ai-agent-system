#  一站式智能 AI Agent 系统

基于 **FastAPI + RAG 检索增强生成 + LangGraph** 搭建的一站式智能 AI Agent 系统。集成知识库问答、工具调用、多轮对话、工作流编排、质量评估等全链路能力，支持本地运行与 Docker 容器化部署，适合智能问答、企业知识库、自动化 Agent 场景落地。

##  核心功能

系统覆盖 LLM 调用、工具智能编排、RAG 高质量问答、多轮对话、Agent 工作流全场景，核心能力如下：

- **基础服务能力**：FastAPI 高性能接口、日志管理、环境配置、Docker 容器化部署
- **通用工具调用**：内置天气查询、数学计算等工具，支持多工具协同调用
- **企业级 RAG 知识库**：PDF 上传解析、智能分块、向量化存储、多路检索、结果优化
- **高质量检索体系**：向量检索 + BM25 混合检索、Cross-Encoder 重排、MMR 多样性去重、HyDE 语义增强、Redis 缓存加速
- **多轮对话系统**：会话管理、上下文记忆、对话压缩、历史记录持久化
- **智能 Agent 编排**：基于 LangGraph 状态机，实现多步骤推理、任务编排、可观测工作流
- **量化评估体系**：支持 HitRate、MRR、NDCG 等检索指标评估，可量化 RAG 问答质量
- **可视化交互**：内置 Web 前端界面，支持在线问答、文件上传、结果溯源

##  快速开始

### 环境要求
- Python 3.11+
- 可选：Docker & Docker Compose（容器部署）

### 本地开发运行

```bash
# 1. 克隆项目
git clone https://github.com/yourname/ai-agent-system.git
cd ai-agent-system

# 2. 创建并激活虚拟环境
python -m venv venv
# Windows
venv\Scripts\activate
# Linux/Mac
source venv/bin/activate

# 3. 安装项目依赖
pip install -r requirements.txt

# 4. 配置环境变量
cp .env.example .env
# 编辑 .env 文件，填入你的 DEEPSEEK_API_KEY 等密钥

# 5. 启动项目服务
python run.py

# 6. 访问服务
# 前端可视化页面：http://localhost:8000
# 自动生成 API 文档：http://localhost:8000/docs
```

### Docker 容器部署

```bash
# 1. 配置环境变量
cp .env.example .env
# 编辑 .env 配置模型密钥、服务参数

# 2. 一键启动所有服务
docker-compose up -d

# 3. 查看运行日志
docker-compose logs -f api

# 4. 停止服务
docker-compose down
```

### 模型本地离线下载

项目支持本地模型离线部署，规避境外网络访问失败、网页解析异常、在线模型加载超时等问题，通过专属脚本一键下载所需向量、重排模型至本地，实现完全离线推理。

```bash
# 执行模型批量下载脚本
python download_model.py
```

**脚本功能说明：**
- 自动下载项目依赖的 Embedding 向量化模型、Cross-Encoder 重排模型
- 默认保存至项目本地模型目录，全局复用，无需重复联网加载
- 规避 GitHub、境外模型源访问失败、解析失败等网络问题
- 下载完成后自动配置本地模型加载路径，服务启动优先读取本地模型

> **注意事项**：首次下载需保证网络通畅，模型文件较大请耐心等待；下载完成后可完全离线运行项目，不受外网限制。

##  核心模块说明

- **混合检索模块**：实现 BM25 关键词检索 + 向量语义检索多路召回，兼顾精准匹配与语义理解
- **结果优化模块**：Cross-Encoder 精准重排 + MMR 多样性去重，解决结果冗余、排序不准问题
- **HyDE 增强模块**：通过假设文档生成，优化短问句、模糊问句的向量检索效果
- **缓存模块**：基于 Redis 缓存检索结果，大幅降低重复请求耗时与模型调用成本
- **RAG 评估模块**：自动化计算 HitRate、MRR、NDCG 指标，量化检索与问答质量
- **LangGraph Agent**：状态机驱动工作流，支持复杂任务多步骤拆解、推理、执行

##  常见问题

1. **检索指标偏低（HitRate、NDCG 分数不高）**
   可通过：优化文本分块大小与重叠度、调整混合检索权重、开启 HyDE 增强、启用重排+MMR 筛选优化效果。
2. **接口响应慢**
   默认开启 Redis 检索缓存，重复问题可秒级响应；首次检索可适当调粗召回数量，平衡速度与精度。
3. **LLM 调用失败**
   检查 `.env` 文件中模型密钥是否正确、接口域名是否可访问、网络是否正常。
